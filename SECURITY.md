# Security Policy

## Teaching Examples & Proof of Concepts
The code, scripts, and materials found in this repository are designed strictly as **teaching examples** and **Proof of Concept (PoC)** demonstrations. They are structured to illustrate specific programming, architectural, or technical concepts in a simplified manner.

**This software is intentionally NOT secured and is not fit for production use.**

## Production Warning
* **Educational Simplification:** To keep the lessons clear and focused, many standard security practices (such as strict input validation, robust error handling, and secure dependency management) have been omitted.
* **No Security Audits:** This codebase has not undergone any security reviews. It contains deliberate gaps, insecure defaults, or potential vulnerabilities meant for learning environments only.
* **Use at Your Own Risk:** Running, deploying, or copying this code into a live, public, or production environment is done entirely at your own risk.

## How to Handle Vulnerabilities
Because these materials serve exclusively as insecure teaching aids, we do not issue traditional security patches. However, if you are a student or developer and notice a security flaw, feel free to open a discussion or issue! Discussing *why* these examples are insecure is an excellent way to further the educational goals of this repository.
